## 5. Daemon Design

### daemon 的主设计是什么
Job 6 的 daemon 是一个**配置驱动的 loop/once 执行器**，而不是 cron wrapper，也不是重型调度平台。

### 主入口
```bash
python -m ifa_data_platform.lowfreq.daemon
```

### 支持的主模式
- `--once`
- `--loop`
- `--health`
- `--config`

### daemon 负责什么
- 读取 daemon config
- 判断当前时间命中哪个 window
- 调用对应 group orchestration
- 记录 daemon / group state
- 输出 summary / logs
- 对外提供 health / status / freshness

### daemon 不负责什么
- 不理解 trade_cal / stock_basic 业务语义
- 不做 facts/signals
- 不做 report
- 不做复杂 scheduler 平台逻辑

### 当前实现方式
通过：

- `GroupStateStore`

### 当前正式主路径

### 当前状态能表达什么
#### daemon 级
- 最近一次 loop 时间
- 最近一次运行 group
- 最近一次运行状态
- 最近一次成功时间

#### group 级
- 最近运行时间
- 最近成功时间
- 最近状态
- retry_count
- is_degraded
- in_fallback

### retry / fallback 逻辑
- 每个 window 有 `max_retries`
- retry_count 达到阈值后：
  - group 标记为 `degraded`
- fallback window 可以再次尝试
- 不允许无限重试
