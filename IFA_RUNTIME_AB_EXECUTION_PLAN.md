# IFA_RUNTIME_AB_EXECUTION_PLAN.md

> 文档性质：低频（A）+ 中频（B）后续工作规划文档。
>
> 文档目标：把 A 类（低频）与 B 类（中频）的后续工作拆成清晰的正式阶段，明确目标、边界、输入、输出、验收标准与非目标。
>
> 约束说明：
> - 本文档只做规划，不进入实现。
> - 本文档不新增具体 dataset，不改 lowfreq / midfreq 代码，不做 migration。
> - 本文档不进入 Archive Layer 实现，不进入 highfreq 实现。

---

## 1. 总体执行原则

当前 A/B 一体化后的执行原则固定为：
1. 先固化设计与规划，再做研发实现。
2. 低频（A）与中频（B）视为同一 Runtime Daemon 体系下的不同工作面。
3. 先做现状审计和对账，再做扩展补齐。
4. 先把运行、监控、summary、watchdog 理清，再继续扩数据面。
5. 当前只聚焦 lowfreq + midfreq，不进入 Archive 实现，不进入 highfreq 实现。

后续固定推进顺序：
- A1 → A2 → A3 → A4 → A5
- B1 → B2 → B3 → B4 → B5 → B6

---

## 2. A 类（低频相关）

### A1：低频现状审计与旧运行链清理
#### 1. 目标
- 审计当前 lowfreq 的真实运行面、真实表面、真实任务面
- 识别旧运行链、遗留注册项、测试项、重复命名项
- 为后续低频扩展与统一 Runtime 理解打基础

#### 2. 边界
- 只做审计、梳理、清理规划
- 不新增具体 dataset
- 不大改 lowfreq 代码逻辑

#### 3. 输入
- 当前 lowfreq daemon / group 配置
- 当前 lowfreq runs / group state / dataset registry
- 当前 lowfreq 文档与 dataset 台账

#### 4. 输出
- 低频现状审计结论
- 旧运行链 / 重复命名 / 测试项清单
- 后续保留 / 隔离 / 清理建议

#### 5. 验收标准
- 能明确说清 lowfreq 真实在跑什么
- 能明确区分生产项与测试项
- 能明确哪些遗留项应后续治理

#### 6. 非目标
- 不直接扩 lowfreq 新数据
- 不直接做 migration
- 不直接重构 lowfreq 框架

---

### A2：低频补宏观数据
#### 1. 目标
- 把宏观慢变量正式补进 lowfreq 设计与实现范围
- 形成宏观低频资产层

#### 2. 边界
- 只做宏观慢变量方向
- 不做窗口化中频宏观
- 不做高频宏观

#### 3. 输入
- 当前 lowfreq 基础框架
- 宏观数据清单（利率、CPI/PPI、PMI、GDP、工业增加值、社融/M2 等）
- A/B 一体化设计文档

#### 4. 输出
- 宏观低频数据范围定义
- 对应 lowfreq 任务设计
- 宏观低频资产层文档与后续实现规划

#### 5. 验收标准
- 宏观低频范围清晰
- 与低频定位一致
- 不与中频宏观窗口职责混淆

#### 6. 非目标
- 不做宏观中频窗口
- 不做 Archive 宏观实现
- 不做高频宏观

---

### A3：低频补商品 / 期货 / 贵金属慢变量
#### 1. 目标
- 把商品 / 期货 / 贵金属的慢变量和基础信息补进 lowfreq 范围
- 形成相应低频基础资产层

#### 2. 边界
- 只做慢变量、基础表、基础参数
- 不做窗口化中频刷新
- 不做分钟级历史层

#### 3. 输入
- 当前 lowfreq 框架
- 商品 / 期货 / 贵金属重点对象范围定义
- A/B 一体化设计基线

#### 4. 输出
- 商品 / 期货 / 贵金属低频范围定义
- 重点对象清单原则
- 对应低频任务规划

#### 5. 验收标准
- 明确哪些属于低频慢变量层
- 明确重点池如何受控
- 不与归档层或中频层混淆

#### 6. 非目标
- 不做 Archive 历史补齐
- 不做分钟历史
- 不做高频/流式数据

---

### A4：低频补科技板块静态清单 / 成分
#### 1. 目标
- 把科技板块专项中的静态层补进 lowfreq
- 建立固定科技主线、板块静态成分、代表股长期清单基础

#### 2. 边界
- 只做静态层、基础层
- 不做动态热点发现
- 不做中频窗口热点追踪

#### 3. 输入
- 科技板块专项设计
- focus tables 设计方向
- lowfreq 基础框架

#### 4. 输出
- 固定科技主线清单定义
- 静态板块 / 成分关系定义
- 长期代表股基础层规划

#### 5. 验收标准
- 科技板块静态层边界清晰
- 能与动态热点层区分
- 能支持后续中频消费

#### 6. 非目标
- 不做动态发现池
- 不做板块窗口刷新
- 不直接做热门池实时更新

---

### A5：低频运行、监控、日报最终收尾
#### 1. 目标
- 把 lowfreq 从“能跑”收成“可长期维护、可监控、可日报”的正式运行态

#### 2. 边界
- 聚焦运行治理与监控收尾
- 不大扩新数据面

#### 3. 输入
- 当前 lowfreq daemon / runs / group state
- A1–A4 的审计与扩展结论
- runbook / dataset registry / 设计文档

#### 4. 输出
- lowfreq 最终运行说明
- 健康检查与 watchdog 说明
- 业务可读日报 / summary 方案
- blocker 识别与排查路径

#### 5. 验收标准
- lowfreq 可查状态
- lowfreq 可查 summary
- lowfreq 健康与问题能被业务可读化表达

#### 6. 非目标
- 不继续扩大量新 dataset
- 不进入 Archive 运行收尾
- 不进入高频实现

---

## 3. B 类（中频相关）

### B1：中频现状审计与 Batch 对账
#### 1. 目标
- 审计当前 midfreq 的真实运行状态、真实 dataset 闭环情况、Batch 进度与遗留问题
- 把“代码推进了”与“真正闭环”区分清楚

#### 2. 边界
- 只做对账和审计
- 不继续扩新数据
- 不立刻开发新 batch

#### 3. 输入
- 当前 midfreq daemon / summary / watchdog
- 当前 midfreq current/history/version 实况
- B4/B5/B6 已有文档

#### 4. 输出
- 中频现状审计结论
- Batch 对账结果
- 当前闭环项 / 未闭环项清单

#### 5. 验收标准
- 能明确说清哪些中频 dataset 真正闭环
- 能明确 blocker 与遗留项
- 能为后续 B2–B6 提供清晰基线

#### 6. 非目标
- 不直接扩数据面
- 不补新 migration
- 不进入高频

---

### B2：中频补股票扩展数据
#### 1. 目标
- 在股票中频层补齐当前明确需要的扩展数据
- 强化股票重点对象与结构化市场状态覆盖

#### 2. 边界
- 只做股票中频扩展面
- 不扩到宏观 / 商品 / 科技动态热点

#### 3. 输入
- 中频现状审计结果
- B Universe 边界
- 重点关注 / watchlist / focus tables 设计方向

#### 4. 输出
- 股票中频扩展需求清单
- 任务拆解与后续实现规划
- 与当前 Batch 的衔接关系

#### 5. 验收标准
- 股票中频扩展范围清晰
- 不越界到其他资产类别
- 与当前 Runtime 统一视角一致

#### 6. 非目标
- 不做宏观中频
- 不做商品中频
- 不做 Archive 股票历史实现

---

### B3：中频补宏观窗口数据
#### 1. 目标
- 明确并补齐适合中频窗口刷新的宏观数据层
- 形成宏观窗口型状态更新面

#### 2. 边界
- 只做窗口型宏观
- 不把慢变量宏观和 Archive 宏观混进来

#### 3. 输入
- 宏观低频范围定义
- 中频窗口设计
- 当前 report-time 对宏观窗口数据的需求

#### 4. 输出
- 中频宏观窗口数据定义
- 对应 group / window 使用逻辑
- 后续实现规划

#### 5. 验收标准
- 能清楚区分低频宏观 vs 中频宏观
- 能明确宏观窗口数据服务什么报告时点
- 不偏向高频实时系统

#### 6. 非目标
- 不做宏观归档
- 不做高频宏观
- 不做大范围重构

---

### B4：中频补商品 / 期货 / 贵金属窗口数据
#### 1. 目标
- 为商品 / 期货 / 贵金属建立窗口型中频数据层
- 支持重点对象的盘前 / 午间 / 收盘后结构化状态刷新

#### 2. 边界
- 只做窗口型更新
- 不做归档历史层
- 不做流式高频层

#### 3. 输入
- 商品 / 期货 / 贵金属重点池定义
- 中频窗口框架
- 当前 Runtime Daemon 统一视角

#### 4. 输出
- 中频商品 / 期货 / 贵金属范围定义
- 窗口型任务规划
- 与 lowfreq 慢变量层的职责分工

#### 5. 验收标准
- 能清楚定义哪些数据属于中频窗口层
- 能与低频和归档层分层明确
- 不突破“中频不是高频流系统”的原则

#### 6. 非目标
- 不做分钟流式采样
- 不做 Archive 历史回填
- 不做高频实现

---

### B5：中频补科技板块动态热点 / 代表股
#### 1. 目标
- 把科技板块专项中的动态发现层和代表股窗口跟踪补进中频
- 形成科技板块的动态热点层

#### 2. 边界
- 只做中频动态热点层
- 不重复做低频静态成分层

#### 3. 输入
- 科技板块专项设计
- focus tables 设计方向
- 常驻池 vs 动态池双层框架

#### 4. 输出
- 科技板块动态热点定义
- 每个热门板块前 10 / 前 20 活跃股的中频规划
- 常驻池与动态池的中频消费方式

#### 5. 验收标准
- 能清楚区分静态层与动态层
- 能说明常驻池和动态池如何共同服务中频
- 不退化成硬编码热门列表

#### 6. 非目标
- 不做 Archive 科技历史层实现
- 不做高频科技热点层
- 不把 focus 逻辑写死在代码常量中

---

### B6：中频运行与验收收尾
#### 1. 目标
- 把中频从“局部能跑”收成“整体可运行、可验收、可长期维护”

#### 2. 边界
- 聚焦运行治理、对账、summary、watchdog、最终验收
- 不继续无节制扩数据面

#### 3. 输入
- B1–B5 的审计与扩展结论
- 当前 midfreq daemon / summary / watchdog 状态
- 现有 runbook 与 dataset registry

#### 4. 输出
- 中频最终运行说明
- 中频验收标准与闭环口径
- 业务可读 summary / 报告方式
- blocker 与排查路径

#### 5. 验收标准
- 中频状态可查
- 中频 summary 可查
- 中频 watchdog 有明确健康口径
- 能明确哪些 dataset 真正闭环、哪些仍未闭环

#### 6. 非目标
- 不进入 Archive 实现
- 不进入 highfreq 实现
- 不把中频变成持续流系统

---

## 4. 一句话执行目标

先把低频（A）与中频（B）作为统一 Runtime Daemon 体系下的两个工作面固化为正式规划，然后按 A1–A5 与 B1–B6 的固定顺序推进：先审计对账，再补低频与中频关键空白，最后完成运行、监控、日报与验收收尾。
